Project Overview
This repository contains the lab environment setup and verification files for the Foundations of Cybersecurity course. The lab was built using Ubuntu Linux running inside a VMware virtual machine on a physical host computer. The environment includes essential tools such as Git, Python 3, Bash, SSH, and curl, all of which were verified using the included lab_verify.sh script.
Repository Contents

lab_verify.sh — Shell script that verifies all required tools are installed and functioning
setup_verify.txt — Output file generated by running lab_verify.sh inside the Ubuntu VM

Security Philosophy
The lab environment built in this course is not just a technical exercise — it is a direct, hands-on application of the foundational principles that govern modern cybersecurity. Specifically, the design and use of this virtualized lab environment connects deeply to the CIA Triad: Confidentiality, Integrity, and Availability.
Confidentiality
Confidentiality refers to ensuring that sensitive information is accessible only to those who are authorized to view it. In this lab, virtualization plays a critical role in maintaining confidentiality. By running Ubuntu inside a VMware virtual machine, the lab environment is fully isolated from the host operating system. This means that any tools, scripts, or files created within the VM cannot be accessed by other users or applications on the physical machine without explicit permission. This isolation mimics how enterprise environments use virtual machines to separate sensitive workloads from general-purpose systems, reducing the risk of unauthorized data exposure. As Stallings (2019) explains, isolation is one of the most effective mechanisms for enforcing confidentiality in computing environments, because it creates clear boundaries between systems that would otherwise share resources.
Integrity
Integrity refers to ensuring that data and systems remain accurate, consistent, and unaltered by unauthorized parties. This lab environment directly supports integrity through the use of Git and GitHub as a version control system. Every change made to the lab files — including the verification script — is tracked with a commit message, a timestamp, and a record of who made the change. This creates an auditable history that can be used to detect unauthorized modifications. If any file were changed without authorization, the Git log would reveal the discrepancy. According to the National Institute of Standards and Technology (NIST, 2020), version control systems are a recognized best practice for maintaining the integrity of software and configuration files in secure environments, because they provide a verifiable chain of custody for all changes.
Availability
Availability refers to ensuring that systems and data are accessible to authorized users when they are needed. This lab supports availability in two key ways. First, by hosting the environment in a virtual machine, the lab can be snapshotted, backed up, and restored quickly if something goes wrong — ensuring that the environment is never permanently lost. Second, by pushing all files to GitHub, the lab environment is accessible from any machine with an internet connection. Even if the physical laptop or VM were to fail, the scripts and documentation stored in the GitHub repository would remain available. This approach reflects the principle that redundancy and cloud-based storage are essential strategies for maintaining availability in modern IT and cybersecurity operations (Stallings, 2019).
Conclusion
Taken together, the use of VMware virtualization for isolation, Git for integrity tracking, and GitHub for redundant availability demonstrates that this lab environment was built with the CIA Triad in mind. These are not abstract concepts — they are active design choices that reflect how real-world cybersecurity professionals think about and build secure systems.
